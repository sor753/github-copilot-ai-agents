# 機能仕様

このドキュメントは DailyDots の主要機能と画面フローを定義します。

## 対象機能

- ホーム
- マイジャーナル（一覧）
- ジャーナル詳細
- 新規作成 / 編集
- 削除

## 画面別仕様

## 1. ホーム（`/`）

### 目的

- 全体状況を短時間で確認する
- 当日分をすぐに記録する

### 表示内容

- 総エントリ数
- 直近7日エントリ数
- 連続記録日数
- 最新の気分
- 今日のクイック追加フォーム

### 操作

- 今日の日付固定で記録を作成または更新
- 「マイジャーナルを開く」から一覧へ遷移

## 2. マイジャーナル（`/journal`）

### 目的

- 過去エントリを一覧で管理する

### 表示内容

- 日付降順のエントリ一覧
- 各エントリの要約情報（気分、本文の一部）

### 操作

- 新規作成画面へ遷移
- 詳細を開く
- 削除

## 3. ジャーナル詳細（`/journal/:date`）

### 目的

- 指定日の記録を詳細確認する

### 表示内容

- 日付
- 気分
- 本文

### 操作

- 編集画面へ遷移
- 一覧へ戻る

## 4. 新規作成 / 編集（`/journal/new`, `/journal/:date/edit`）

### 目的

- 記録の作成または更新を行う

### 入力項目

- 日付
- 気分
- 本文

### 挙動

- 同日付の記録が存在する場合は上書き（upsert）
- 本文入力中、文字数をリアルタイムで表示
- 保存成功後は詳細画面へ遷移

## 5. Not Found（`/404`）

### 目的

- 不正なパスのアクセス時に案内する

## 主要フロー

## 作成フロー

1. 新規作成画面を開く
2. 日付・気分・本文を入力
3. 保存を実行
4. 成功後、詳細画面へ遷移

## 更新フロー

1. 詳細画面またはホームから編集に進む
2. 内容を変更して保存
3. 対象日付の記録を更新

## 削除フロー

1. 一覧画面で削除操作を実行
2. 対象日付の記録を削除
3. 一覧とサマリーを再取得

## エラー時の振る舞い

- データ取得失敗時は画面上にエラーメッセージを表示
- 対象日付が存在しない場合、詳細画面で該当なしメッセージを表示

## 非機能的な前提

- 永続化先は localStorage
- ネットワーク接続なしで動作可能
- 将来的に services 層差し替えで Supabase 移行可能
